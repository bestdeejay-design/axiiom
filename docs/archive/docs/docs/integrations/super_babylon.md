# Интеграция с Super Babylon

> **Источники:**
> - `references/github/store-apis-research.md` — исследование
> - `SRS/db/schema.rb` — `shop_group_enum: babylon`

## Общие сведения

| Параметр | Значение |
|----------|----------|
| **Тип интеграции** | Без API (физическое присутствие сборщика) |
| **Протокол** | Отсутствует |
| **Формат данных** | Парсинг сайта / XLS-прайсы для каталога |

## Статус интеграции

Super Babylon — **особый случай**. У сети нет публичного API для партнёров. Сборка заказов происходит физически: пикеры сервиса приходят в магазин и собирают заказы с полок.

- `shop_group_enum` включает `babylon`
- В коде адаптер: `BabylonAdapter` (парсинг / XLS)
- Каталог обновляется через загрузку прайс-листов (XLS)
- Цены и остатки — парсинг сайта superbabylon.ru

## Процесс сборки

1. Заказ поступает в систему
2. Пикер получает задание (через Picker API)
3. Пикер физически идёт в магазин Super Babylon
4. Собирает товары с полок
5. Отмечает собранные позиции в приложении
6. Передаёт заказ курьеру

## Особенности для реализации

- **Нет API** для создания заказов, проверки остатков, получения цен в реальном времени
- Цены и ассортимент — **нестабильные** (могут меняться без уведомления)
- Рекомендуется: парсинг сайта с кэшированием (TTL 1 час)
- Для MVP можно использовать Super Babylon только как дополнительную сеть (V2)
- Физическая сборка накладывает ограничения на SLA (время сборки + дорога до магазина)
