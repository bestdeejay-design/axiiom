ДЕТАЛЬНЫЙ ПЛАН ДЕМО-ВЕРСИИ PADELPRO v2
=========================================

0. КОНЦЕПЦИЯ И УРОВЕНЬ КАЧЕСТВА
   Цель: демо должно выглядеть как настоящий SaaS-продукт уровня Playtomic/Cue,
   а не как учебный проект. Владелец клуба должен захотеть это купить.

   Ориентиры по качеству:
   - Профессиональная вёрстка: сайдбар, хлебные крошки, карточки, графики
   - Продуманные микровзаимодействия: анимации, переходы, лоадеры, тосты
   - Типографика: иерархия заголовков, читаемые таблицы с выравниванием сумм
   - Мобильная версия не «растянутая десктопная», а перестроенная под телефон
   - Цветовая схема «спорт/бизнес»: тёмная тема или чистый светлый CRM-стиль

1. ТЕХНОЛОГИЧЕСКИЙ СТЕК (расширенный)
   1.1. Frontend: HTML5, CSS3 (Custom Properties, Grid, Flexbox), ES6+ JavaScript
   1.2. Библиотеки:
        - Chart.js 4.x — все графики (P&L, загрузка кортов, зарплаты)
        - QRCode.js — генерация QR
        - jsQR — сканирование QR
        - Font Awesome 6 — иконки
        - Google Fonts: Inter (основной) + Montserrat (заголовки)
   1.3. Хранение: localStorage (многоуровневое: демо-данные + сессия + настройки)
   1.4. Хостинг: GitHub Pages

2. АРХИТЕКТУРА ПРИЛОЖЕНИЯ
   2.1. Single Page Application с хеш-роутингом (#/client, #/trainer, #/admin, #/login)
   2.2. Компонентный подход без фреймворка:
        - Каждый экран — функция render*(), возвращающая HTML-строку
        - Переиспользуемые UI-компоненты: Modal, Toast, Tabs, DataTable, StatCard, ChartCard
   2.3. Слой данных:
        - DataService — единая точка работы с localStorage
        - Модели с валидацией и методами
        - Начальное заполнение демо-данными с генерацией истории за 30 дней
   2.4. EventBus — централизованная система событий для обновления UI

6. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ ПО РОЛЯМ (детально)

   6.1. СТАРТОВАЯ СТРАНИЦА (мини-презентация продукта)
        - Крупный логотип «PadelPro» + подзаголовок «Управление падел-клубом»
        - Короткий продающий текст: «Первый специализированный SaaS для падел-клубов в России и СНГ. Автоматизация финансов, контроль тренеров, аналитика кортов.»
        - Три ключевых преимущества иконками: P&L в реальном времени, Честный корт (QR+камера), Авторасчёт зарплат
        - Три крупные карточки-кнопки для входа в демо:
          * «Клиент» — запись, абонементы, сканер QR
          * «Тренер» — расписание, генерация QR, зарплата
          * «Администратор» — дашборд, сетка расписания, аналитика, экспорт
        - Под каждой карточкой — готовая демо-подсказка с логином
        - Внизу: «Демо-версия • 50 000 ₽/мес • Пробный период 1–2 месяца бесплатно»
        - Адаптив: на мобильных карточки в столбик

   6.2. КЛИЕНТ (Client Dashboard)
        6.2.1. Верхняя панель:
              - Имя клиента, аватар
              - Виджет абонемента: тип, остаток, прогресс-бар, дата окончания
              - Кнопка «Пополнить абонемент» (мок)
         6.2.2. Список тренировок (основной экран):
               - Вкладки: Предстоящие | История
               - Каждая запись — карточка с: дата, время, тренер, корт, статус
               - Для scheduled: кнопка «Отменить»
               - Для in_progress: подсказка «Ожидает подтверждения»
               - Пустое состояние: иллюстрация + призыв записаться
        6.2.3. Запись на тренировку:
              - Шаг 1: Выбор даты (календарь на 7 дней)
              - Шаг 2: Выбор времени (сетка доступных слотов)
              - Шаг 3: Выбор тренера (карточки с фото-плейсхолдером, специализацией)
              - Шаг 4: Выбор корта (карточки с типом и ценой)
              - Шаг 5: Подтверждение — сводка и кнопка «Записаться»
        6.2.4. Сканер QR:
              - Полноэкранный режим сканера
              - Визуальная рамка-прицел
              - При успехе: зелёная анимация + «Тренировка подтверждена! Списано 1 занятие»
              - При ошибке: красная подсветка + текст ошибки
         6.2.5. Профиль:
               - Имя, телефон, аватар
               - Статистика: тип абонемента, остаток, всего посещений
               - Срок действия абонемента

   6.3. ТРЕНЕР (Trainer Dashboard)
        6.3.1. Верхняя панель:
              - Имя, код, ставка
              - Статистика месяца: проведено часов, заработано
        6.3.2. Расписание (основной экран):
              - Вкладки: Сегодня | Завтра | Неделя
              - Таймлайн тренировок с цветовой кодировкой по статусу
              - Для каждой: время, клиент, корт, статус
        6.3.3. Действия с тренировкой:
              - «Начать» → генерируется QR, статус → in_progress, запускается таймер
              - Пока идёт: показывается QR-код + таймер длительности
              - «Завершить» → статус → completed, считается оплата тренеру
              - Если клиент не отсканировал QR → тренировка завершается, но помечается как «без подтверждения» → алерт админу
        6.3.4. QR-модалка:
              - Крупный QR-код с данными тренировки
              - Таймер действия QR (5 минут) — по истечении сгенерировать новый
              - Кнопка «Обновить QR»
        6.3.5. Моя зарплата:
              - Таблица завершённых тренировок с суммой
              - Итоговая сумма за месяц
              - Кнопка «Скачать ведомость» (CSV)

   6.4. АДМИНИСТРАТОР / ВЛАДЕЛЕЦ (Admin Dashboard)
        6.4.1. Главный дашборд (P&L):
              - Верхний ряд (StatCard): Выручка сегодня | Выручка месяц | Расходы месяц | Чистая прибыль
              - Каждая StatCard: значение, подпись, тренд (▲▼ с %), цвет (зелёный/красный)
              - График выручки за 30 дней (Chart.js: линейный)
              - График «Выручка по кортам» (Chart.js: столбчатый)
              - График «Загрузка кортов» (Chart.js: горизонтальные бары, %)
              - Сводка дня: всего тренировок, проведено, отменено, активные
        6.4.2. Управление тренерами:
              - Таблица: имя, ставка, часы за месяц, начислено, статус
              - Ведомость зарплаты: кнопка «Сформировать» → таблица + экспорт CSV
              - График «Зарплаты тренеров» (Chart.js: столбчатый)
        6.4.3. Все тренировки:
              - Полноценная таблица с: поиск, фильтр по дате (календарь), фильтр по тренеру, фильтр по статусу
              - Сортировка по всем колонкам
              - Пагинация (по 15 записей)
              - Экспорт в CSV (с учётом фильтров)
        6.4.4. Аналитика кортов:
              - Таблица: Корт | Загрузка % | Выручка | Часы пик | Простой
              - График загрузки по дням недели и часам (тепловая карта или таблица)
              - Рекомендации: «Корт №3 простаивает во вторник утром — скидка 20%»
        6.4.5. Финансы (P&L детально):
              - Таблица доходов/расходов по дням
              - Круговой график «Структура расходов»
              - Ключевые метрики: средний чек, выручка на корт, маржинальность

         6.4.6. СЕТКА РАСПИСАНИЯ ВСЕХ ТРЕНЕРОВ (Playtomic-style):
               - Ключевой экран для демо — владелец видит ВСЁ расписание на одном экране
               - Строки = корты (4 корта), колонки = временные слоты (08:00–22:00, шаг 1 час)
               - В каждой ячейке: карточка тренировки (тренер + клиент + статус)
               - Цвет ячейки по статусу: свободно (серый), запланировано (синий), идёт (зелёный), завершено (тёмный)
               - Переключение дней: стрелки ← сегодня →, быстрые даты
               - Клик по ячейке: модалка с деталями тренировки
               - Клик по пустой ячейке: быстрое создание тренировки
               - В мобильной версии: вертикальный таймлайн вместо сетки

7. ЛУЧШИЕ ПРАКТИКИ CRM (контрольный список)
   7.1. Навигация:

   7.3. Таблицы:
        - Сортировка по клику на заголовок (▲▼)
        - Поиск с debounce 300ms
        - Пагинация: «1–15 из 60»
        - Пустое состояние: иконка + «Нет данных» + CTA-кнопка
   7.4. Формы:
        - Валидация на blur и на submit
        - Подсветка ошибочных полей красным
        - Маски для телефона (+7 900 123-45-67)
   7.5. Мобильная версия:
        - Нижняя навигационная панель (4–5 иконок)
        - Таблицы → карточки
        - Кнопки действий во всю ширину

   7.6. Демо-режим:
        - Баннер «ДЕМО-ВЕРСИЯ» над сайдбаром

9. КРИТЕРИИ ПРИЁМКИ
   9.1. Внешний вид: неотличим от коммерческого SaaS (Playtomic, Cue)
   9.2. Все 5 модулей из контекста имеют визуальное представление в демо
   9.3. Сетка расписания — центральный экран, работает быстро и наглядно
   9.4. Сквозной сценарий записи и QR-подтверждения работает без ошибок
   9.5. Дашборд админа показывает осмысленные данные, а не «рыбу»
   9.6. Данные сохраняются в localStorage и переживают перезагрузку
   9.7. QR-сканер работает через камеру на мобильных
   9.8. Экспорт CSV корректен (UTF-8 BOM, разделители)
   9.9. Все графики Chart.js отрисовываются корректно
   9.10. Мобильная версия полностью функциональна
   9.11. Время загрузки < 3 секунд (CDN)
